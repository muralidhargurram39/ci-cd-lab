# Security Policy

## Supported Versions

| Version | Supported |
|----------|-----------|
| 1.x | ✅ |
| < 1.0 | ❌ |

---

## Reporting a Vulnerability

If you discover a security issue in this project:

- Do not publish the vulnerability publicly.
- Create a GitHub Issue with minimal technical details, or contact the maintainer directly.
- Include steps to reproduce the issue, affected components, and any relevant logs.

The issue will be reviewed and acknowledged as quickly as possible.

---

## Security Best Practices

This project follows these practices:

- Principle of Least Privilege
- Secure Docker Images
- Persistent Data Isolation
- Version-controlled Infrastructure
- Static Code Analysis with SonarQube
- Artifact Repository Management with Nexus
- Containerized Service Isolation
